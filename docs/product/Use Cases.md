# StackAlchemist: Product Use Cases

StackAlchemist is designed to eliminate the "boilerplate tax" for developers and founders. Below are the primary use cases for the platform.

## 1. The Solo Founder (MVP Speedrun)
*   **Persona:** A non-technical or semi-technical founder with a validated business idea.
*   **Use Case:** The founder uses **Simple Mode** to describe their idea (e.g., "A subscription-based marketplace for freelance photographers").
*   **Outcome:** StackAlchemist generates a fully functional .NET/Next.js stack with Stripe integrated, allowing the founder to launch and start taking payments in hours instead of weeks.

## 2. The Agency Developer (Efficiency at Scale)
*   **Persona:** A developer at a software agency building dozens of client MVPs per year.
*   **Use Case:** The developer uses **Advanced Mode** to precisely define complex entity relationships and API endpoints for a new client project.
*   **Outcome:** They receive a production-ready codebase that follows elite architectural standards (SOLID, DRY, Dapper), which they can then hand off to their junior devs to fill in the custom business logic.

## 3. The Enterprise Architect (Standardization)
*   **Persona:** A Senior Architect at a mid-to-large company looking to standardize the tech stack across internal teams.
*   **Use Case:** Uses StackAlchemist to generate "Golden Path" repositories that ensure every new internal tool uses the same .NET 10 / Supabase / Next.js architecture.
*   **Outcome:** Drastically reduced maintenance costs and easier cross-team collaboration because every project follows the same structure.

## 4. The Cloud-Native Engineer (Infrastructure as Code)
*   **Persona:** A DevOps-focused engineer who needs to move from code to a scaled cloud environment.
*   **Use Case:** Purchases **Tier 3 (Infrastructure)** to receive customized AWS CDK scripts and Helm Charts tailored specifically to their generated application.
*   **Outcome:** One-command deployment to AWS or Kubernetes, eliminating the need to manually configure CIDR blocks, VPCs, or load balancers.

## 5. The "Learning by Doing" Developer (Educational)
*   **Persona:** A developer looking to transition from legacy stacks to the modern .NET/Next.js ecosystem.
*   **Use Case:** Generates a project to study how elite-level engineers structure high-scale C# APIs and React frontends.
*   **Outcome:** A high-fidelity "Reference Architecture" that they can fork and modify, serving as a live textbook for modern software design patterns.
